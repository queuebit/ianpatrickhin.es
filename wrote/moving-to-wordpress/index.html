Moving to WordPress

After giving it a lot of thought I decided on Thursday to port IanHines.com from Tumblr to WordPress. After giving it a lot of thought I decided on Thursday to port IanHines.com from Tumblr to WordPress. (I use the term “port” lightly, because there was absolutely nothing straightforward to the process and it certainly was not as simple as exporting and importing entries.) My hope was that this change would be so transparent that the end-user (i.e., you) would not notice any difference.

First of all, why move?

Don’t get me wrong: Tumblr is an exceptionally nice blogging platform. It’s fun to use, has an attractive backend, is sufficiently (and easily) customizable for most users, and makes it incredibly easy to input. Tumblr is a blogger’s “anything bucket,” but it’s not for everyone. Turns out it’s not for me.

In my opinion there are only a few drawbacks associated with Tumblr, but they are significant. The first, and in my view most important, drawback of Tumblr is its inability to export entries using an industry standard format. I know about the Backup Utility: it’s great for storing permanent copies of your data on your hard drive but awful if you want to leave Tumblr for another CMS (like WordPress or ExpressionEngine). CMSs get change over time—for better or worse—sometimes leaving their users seeking another option (see The Great Movable Type Exodus of 2004), and I didn’t like feeling as though I was trapped in Tumblr.

The second reason was more cosmetic: I wanted more granular control over the theme. Specifically, I wanted to be able to format the archives. In Tumblr, all blogs are forced to use a standardized archives page and, well, it’s ugly. Using WordPress allows me to theme my archives, making the entire site more cohesive and, I think, professional.

How I Did It

Because of the limitations of Tumblr—specifically its inability to export entries in an importable format—moving the blog to WordPress was a very, very onerous process. Here’s how I did it:

  1. I installed WordPress at a new subdomain, blog.ianhines.com, using (mt) Media Temple’s One-Click Installation. Piece of cake.
  2. Configured WordPress’s settings, particularly Permalink Settings so that new entries would render with the proper URLs.
  3. Created a spreadsheet to track URL changes.
  4. Created a corresponding Category in WordPress for each content-type in Tumblr: Text, Links, Photographs, Videos, etc.
  5. Manually re-entered each individual post into WordPress, back-dating them so that they appear to have been posted on their original date. Using the spreadsheet created in Step #3, I tracked each URL change (Old (Tumblr) URL → New (WordPress) URL).
  6. Reformatted my Tumblr theme to work with WordPress. I started with the Starkers theme by Elliot Jay Stocks (Thanks Kyle for that advice!) and I have to say this was the easiest time building a WordPress theme I’ve ever had.
  7. Lastly, using the spreadsheet from Steps #3 & #5, I made a extraordinarily long .htacces file manually redirecting each individual post from their Tumblr URLs to their new, cleaner WordPress URLs. This part, in particular, was quite tedious—but also incredibly rewarding.

The result should be a blog that was seamlessly moved from one URL & CMS to another. There were only two elements that did not make the move gracefully:

  1. The comments, which I could not move.
  2. The Feedburner Feed, which republished all the most recent entries when I moved its source from Tumblr to WordPress. Apologies to my subscribers for the duplications; I hope it wasn’t too big an inconvenience.

When all is said and done, though, this was a change that was made for my benefit, not yours. You really shouldn’t see much in terms of outward difference. The backend, however, has changed dramatically and for the better.
