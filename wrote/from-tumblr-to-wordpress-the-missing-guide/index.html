From Tumblr to WordPress: The Missing Guide

With the recent Tumblr downtime incident fresh in everyone’s minds a lot of folks have, understandably, been considering moving from Tumblr to WordPress. I’ve made that journey twice and I can tell you that it is possible to move your site from Tumblr to WordPress without any significant front-end changes.

Let me show you how.

Getting Started: What You’ll Need

Before we get started, lets make sure we have all the required ingredients.

You’ll need:

  • a Tumblr blog;
  • a domain name and web hosting;
  • the latest version of WordPress;
  • a spreadsheet;
  • an HTML editor;
  • a CSS editor;
  • an FTP client;
  • patience; and
  • time.

Let’s look at each of these a little more closely.

A Tumblr Blog

This seems self explanatory, but I included it for the sake of being thorough. What is, perhaps, also worth noting is that you’ll need consistent and frequent access to the Dashboard. Best of luck there.

A Domain Name and Web Hosting

I strongly recommend hosting your website with (mt) Media Temple. Their hosting plans are reliable, scalable, and each account comes with a free web domain registration. The majority of websites would be well served by their (gs) Grid-Server hosting plan, but if the $20/month price tag seems a bit steep you may want to consider gifting yourself a year of the (gs) Grid-Server Lite plan.

That said, Media Temple is not a necessity here. Any host that is compatible with WordPress will do.

The Latest Version of WordPress

The latest version of WordPress can be found here. If you are hosting with Media Temple you should take advantage of their 1-Click Installation feature.

A Spreadsheet

We’ll be using this to track our old (Tumblr) URLs to our new (WordPress) URLs. Seriously. It’s tedious, but the results are well worth it.

Excel or Numbers will both work, but frankly I’d recommend using Google Docs here. It’s fast, it auto-saves, and it has all the necessary features.

An HTML Editor and a CSS Editor

It’s likely that if you’re reading this guide you already own these tools. But for those of you that don’t, they’ll be essentially when the time comes for building our WordPress theme. I highly recommend Coda by Panic.

A File Transfer Protocol (FTP) Client

Along the same lines as the HTML & CSS Editors, you probably already have one of these. If not, I recommend Transmit by Panic.

Patience & Time

Make no mistake: this is a time consuming process. I believe whole heartedly that it’s worth the investment of time and energy, but it’s most certainly not a quick fix. Be prepared to settle in for a few days of serious work.

Step One: Install WordPress

Install WordPress on your newly purchased domain. Directions can be found here.

Step Two: Create Categories

Before we even start dealing with moving posts from Tumblr, we need to get WordPress configured properly. We’ll begin by replicating Tumblr’s post types using WordPress’s Categories.

You’ll want to create seven categories. Only seven. They are the seven post types offered by Tumblr:

  • Text
  • Photo
  • Quote
  • Link
  • Chat
  • Audio
  • Video

We’ll be using these categories to distinguish different post types in our theme, so it’s important to be sure to write the “slug” for each category with those exact title (but in lower case). This will allow us to render them distinctly from one another, creating that “mixed format” feel for which Tumblr is so well known.

Step Three: Install the DF-Style Linked List Plugin for WordPress

This plugin, which can be found here, allows you to create Daring Fireball—or Tumblr—style link posts. This is how we achieve the functionality of having the entry’s title link to the target URL, both on the blog and in the RSS feed.

Be sure to read the directions for this plugin and follow them very closely. It does all the heavy lifting for you.

Step Four: Create Custom Fields

In order to input the format-specific content in a way that’s forward compatible we’ll be using Custom Fields. Custom Fields allow us to input specific information in excess of the “Body” of the post, which we can then integrate into our theme creatively.

We’ll need five custom fields:

  • photo
  • quote
  • link_list_url (See Step Three above)
  • audio
  • video

To add a custom field, follow these easy steps:

  1. Open the “Add New Post” page in your WordPress Dashboard.
  2. Scroll down the page to the Custom Fields menu.
  3. Under the “Add New Custom Field” menu, click “Enter New.”
  4. Enter the name of the Custom Field (see above) in the lefthand box. Enter any value in the righthand box (it doesn’t matter at this point).
  5. Click “Add Custom Field.”

The values you assign these Custom Fields do not matter at this point: only the names, or “keys.”

Step Five: Import Tumblr Posts

Now we’re ready to begin the process of moving your entries from Tumblr to WordPress. Fair warning: if you have a lot of content on Tumblr this may be a painfully long process, as Tumblr doesn’t provide a mechanism for exporting your content in a WordPress compatible format. We’ll be doing this manually, one post at a time.

For the sake of my own sanity, I began this process with the most recent entry and worked my way backwards. You might choose to do the opposite. Whatever works for you.

Importing a Text Post

Text posts are the easiest to import. Here’s how you do it:

  1. In Tumblr’s Dashboard, click the “edit” link on the post you wish to import.
  2. In your WordPress Dashboard, click “New Post” in the top left corner.
  3. Copy the title and body of your post from Tumblr to the appropriate WordPress fields.
  4. Manually reset the publish date of the WordPress post to the Tumblr posts publish date by clicking “Edit” next to the words “Publish Immediately.”
  5. Select “Text” as the post’s category (see Step One above). Do not select any other category. Each post can only have one category selected; if you’d like to note the topic of the post please do so using WordPress’ tags.
  6. Click “Publish.”

Importing a Photo Post

With photos we’ll begin using Custom Fields. Here’s how you do it:

    1. In Tumblr’s Dashboard, click the “edit” link on the post you wish to import.
    2. In your WordPress Dashboard, click “New Post” in the top left corner.
    3. Copy the title of your post from Tumblr to the WordPress title field.
    4. Copy the caption of your post from Tumblr to WordPress body field.
    5. Open the entry in Safari or the browser of your choice. View the source.
    6. Highlight and copy the HTML for the photograph, including the link surrounding it (if applicable).
    7. In WordPress, create a custom photo field using the drop down “Custom Fields” selector (see Step Three above).
    8. In the “value,” or righthand, box of the photo field input the HTML you copied from the Tumblr entry’s source code in the following format.

<a href="http://destination.link" title="The Photo Title"><img src="http://image.source" alt="The Photo Title" /><a>

  2. Select “Photo” as the post’s category (see Step One). Do not select any other category. Each post can only have one category selected; if you’d like to note the topic of the post please do so using WordPress’ tags.
  3. Click “Publish.”

  6. In the “value,” or righthand, box of the quote field input the quote from your Tumblr Dashboard.
  7. Manually reset the publish date of the WordPress post to the Tumblr posts publish date by clicking “Edit” next to the words “Publish Immediately.”
  8. Select “Quote” as the post’s category (see Step One). Do not select any other category. Each post can only have one category selected; if you’d like to note the topic of the post please do so using WordPress’ tags.
  9. Click “Publish.”

Importing a Link Post

  1. In Tumblr’s Dashboard, click the “edit” link on the post you wish to import.
  2. In your WordPress Dashboard, click “New Post” in the top left corner.
  3. Copy the title of your post from Tumblr to the WordPress title field.
  4. Copy the caption of your post from Tumblr to WordPress body field.
  5. In WordPress, create a custom link_list_item field using the drop down “Custom Fields” selector (see Step Three above).
  6. In the “value,” or righthand, box of the linked_list_item field input the destination URL of the link from the Tumblr Dashboard.

  9. Click “Publish.”

Frankly, I don’t use either the Audio or Chat post types, so I’m not sure how I would import those sorts of posts. But I think at this point you have an idea of what my approach looks like, and you can likely come up with a like-minded solution.

Note: It’s a good idea to import a few posts—perhaps one of each post type—to WordPress before beginning Step 5. Doing so will ensure that you have accurate entries with which to test your theme.

Be cautious, however, not to invest too much time importing posts before testing your theme, lest you need to change how you are inputting the content to suit your theme.

Step Six: Port the Tumblr Theme to WordPress

I’m not going to spend too much time here explaining how to make a WordPress theme. There are many other great “How-To” sites on that topic to warrant the time. What I will explain is how I use The Loop to format individual entries specific to their type.

Using The Loop

We’ll begin The Loop like we would on any other WordPress blog:

<?php if (have_posts()) : ?>

<?php query_posts(‘showposts=15’); while (have_posts()) : the_post(); ?>

This essentially begins the loop and instructs WordPress to call for the most recent fifteen posts.

We then begin the fun part. We’re going to use if/else statements to specify different formats for posts depending on which category they belong to. This section of The Loop will look roughly like this:

<?php if (in_category('text')) : ?>

Formatting of text posts

<?php elseif (in_category(‘photo’)) : ?>

Formatting of photo posts

<?php elseif (in_category(‘quote’)) : ?>

Formatting of quote posts

<?php elseif (in_category(‘link’)) : ?>

Formatting of link posts

<?php elseif (in_category(‘video’)) : ?>

Formatting of video posts

<?php else : ?>

Default formatting for posts with no category

You can, of course, include as many different post categories as you like. But this basic setup will allow you to use any HTML and CSS trickery you like to format each post type distinctly from the others.

Formatting Link Posts

To take advantage of the DF-Style Linked List plugin we’re going to have to put some very basic formatting into our theme. In order to render the link to an external site, we’ll use the following code.

<a href="<?php the_linked_list_link(); ?>" title="Link to <?php the_title_attribute(); ?>"><?php the_title(); ?></a>

This will create a link whose text is the entry title, but whose URL will take you to the referenced post, not the entry’s permalink. This is the same, in effect, as Tumblr’s link posts.

Accessing Custom Fields

In order to access the photographs, videos, and quotes we’ve inputted in our custom fields, we’re going to need some custom code. In the code below, the word “key” should be replaced with whatever the key—or name—of the custom field your referencing is.

<?php $key = get_post_meta($post->ID, 'key', true); if ($key == '') { ?><?php } else { ?><p><?php echo $key; ?></p><?php } ?>

This code will render the contents of the custom field inside <p> tags, and can be used anywhere inside The Loop. The <p> tags can, of course, be replaced with any other HTML. If the custom field has no content, nothing is displayed.

In practice, I use this code within The Loop within each category section, as appropriate. For example, in the Quotes section I use this code to input the quote text, replacing “key” with “quote.” Similarly in the photos section, etc.

The advantage to using this approach is that, if you decide to drastically change the formatting of content in future redesigns, your individual entries are not hard-coded with design-specific tags. It dramatically reduces forwards compatibility issues.

Step Six: Mapping URL Changes

There’s no good way to automatically forward Tumblr URLs to their newer, cleaner WordPress counterparts. You have to do it with .htaccess.

What do I mean? Tumblr renders its permalink URLs in a long, ugly format that allows for great scalability but little customizability. They look like this:

http://blog.ianhines.com/post/1611339199/20-things

WordPress, on the other hand, affords you great control in how your permalink URLs are displayed. For example, mine are formatted to show the year, month, and entry “slug,” like this:

http://log.ianhines.com/2010/11/twenty-things/

When you move from Tumblr to WordPress your old, Tumblr permalinks will thereafter be broken. At best they’ll take visitors to your old Tumblr blog. At worst they’ll be entirely broken (if you delete the Tumblr account, for example). That is, unless you map them with .htaccess.

But how do you keep track of dozens, or hundreds of individual URLs and map them properly? You use a spreadsheet.

  • In Column A list the old Tumblr URLs.
  • In Column B list the corresponding WordPress URLs.
  • In Column C note whether this page has yet been entered into your .htaccess file.

Within the .htaccess file the code should look like this, with one line per entry (This example shows the .htaccess mapping for the URLs used in the example above. It was entered in the .htaccess file at the root of http://blog.ianhines.com.). Every space is important and must be done precisely.

Redirect 301 /post/1611339199/20-things http://log.ianhines.com/2010/11/twenty-things/

If it sounds rudimentary, it’s because it is. It’s not an elegant solution: it’s a functional one. Importing entries this way is slow and tedious, but the end result is worthwhile. Or, at least, it was for me.

Update (12/08/2010): Consider using the Simple 301 WordPress plugin rather than editing .htaccess directly.

Step Seven: Miscellaneous Thoughts

  • If you’re currently using a custom domain with Tumblr via A-Records, you’ll need to disable that A-Record redirect before trying to access any WordPress pages on that URL. You’ll be able to upload them via FTP with A-Records enabled, but until you remove that record from your DNS every address on that URL will be redirected back to Tumblr. This does not affect subdomains.
  • If you use a custom domain name on Tumblr, be sure to change the custom URL that your Tumblr account masks itself as in the Tumblr Customize page once you’ve finished your migration to WordPress. DO NOT change the A-Records for your WordPress URL, as Tumblr instructs. If you change Tumblr’s settings, but leave the A-Record unchange, Tumblr will simply redirect your username.tumblr.com address to your new blog. For example, click: ianhines.tumblr.com.

That’s It!

Don’t let the length of this post fool you: moving your Tumblr blog to WordPress isn’t particularly difficult—just time consuming. But remember, that’s no discredit to WordPress, as we’re really flexing WordPress’s muscle here to create a tumblelog where one wasn’t intended.

This isn’t a process for everyone—I know that. But if you’re like me, and are willing to add a bit of friction to your work-flow in exchange for reliability and control, then I strongly suggest you spend the time to move to WordPress. In the end you’ll have a weblog with the same multi-format functionality as Tumblr, and the added benefits of self-hosting and all that WordPress has to offer.
